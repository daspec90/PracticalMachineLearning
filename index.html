Exploratory Data Analysis

After reading it in, I looked at the data in an effort to decide which variables and algorithm(s) to use for this project. Although there were 160 variables in the original training set, many of these were mostly NA’s, and I eliminated them, bringing the number of potentially usable variables rapidly down to 60. I wanted to try to make a general prediction model and realized that using a combination of the cvtd_timestamp and the num_window variables could easily predict the outcome of the test set without really any modeling. So, I decided to eliminate those two variables from my analysis, and in fact I ignored the first 8 variables including the names of the individuals, leaving only 52 numeric predictors and the classe outcome in the training set.

Because all of the remaining variables were numeric, I was thinking that I might try to use a more straightforward algorithm such as “glm”. So my first inclination was to look for “bad” data, or highly correlated data, and eliminate it.

Nine of the variables had more than five data points that were greater than four standard deviations from the mean, and at first I considered leaving these variables out of the analysis.

I also was thinking that I might want to do principal component analysis, and for that I might need to transform the data to make it look more like a normal distribution. For this, I made a Q-Q plots like these for the variables total_accel_arm and roll_belt, respectively.

Hmm…not really what I expected. The data for the variable total_accel_arm does generally follow the diagonal and looks normally distributed, but the roll_belt data is very jaggedy. Wait, maybe each jagged part of the plot is associated with a different classe? I made some additional plots of the roll_belt data, by classe, shown below:

library(ggplot2)
par(mar=c(4,4,.1,.1),cex.lab=.95,cex.axis=.9,mgp=c(2,.7,0),tcl=-.3)

ggplot(training, aes(x=classe, y = roll_belt, color=classe))+geom_point(position=position_jitter(width=0.2), alpha=0.05)

It is really pretty amazing how plotting the same data in a different way can affect your interpretation of what might be going on. By looking at the Q-Q plot for roll_belt, I thought that maybe this could be a great predictor and that each segment or break in the line might represent a different class. The boxplot told a different story: B,C,D and E classe looked pretty much the same. Maybe the A classe was different (the mean surely was), but the spread of data looked similar. But then when I made the scatterplots, a whole different picture emerged. This is highly discontinuous data. Those jagged parts of the Q-Q plot do not exclusively represent one class. Yet, there were clearly distinct (and unambiguous) differences between the class categories.

I looked at scatterplots for most of the variables. Variables with **_belt** in their name seemed to distinguish categories D & E from A, B, and C. Variables with **_arm** distinguished A. **_dumbell** seemed to differentiate C, B and A from the others. And **_forearm** seemed to differentiate A. These were just general impressions from looking at the patterns.

I made a correlation matrix, and there were numerous highly positively correlated variables, but as a first try, I did not eliminate any of the variables, even the ones with the extreme outliers (the outliers seemed to be associated with one class, which could help for classification). I decided to try the “gbm” and “rf” algorithms, because they seemed best suited for this type of data. I chose not to do principal component analysis.

Building and testing prediction models.

First, I randomly partitioned the training set into validation, testing and training sets.

I used the train.set to build a model using the random forest algorithm (“rf”) in the caret package.

# Build a "rf" model. Do not use formula interface (takes much longer).
library(caret)
predictors <- names(train.set[, 8:59])

set.seed(569)
rf.mod <- train(y=train.set$classe, x=train.set[,predictors], method="rf", data=train.set, trControl=trainControl(method="cv"), number=3)

As the results above show, this model has worked pretty well. The OOB (out-of-bag) error estimate is only 1.1%. During the construction of each tree, a bootstrap sample (with replacement) is used to construct the tree. The unselected samples (OOB) are then used to test the accuracy of that tree (and I assume all of the error estimates for all of the trees are averaged). I found it interesting that the lowest estimated error was for class E, which by eye looked to me to be the most different from the others.

To get an unbiased estimate of the accuracy, I used this model to predict classes on the test.set data I set aside previously. Then I made a confusionMatrix to compare the predicted classes to the known classes in the test.set.

There are other things that I could do to attempt to improve this model, but I honestly do not think that it is worth it. I am open to your suggestions. I did use the “gbm” algorithm on the same data set. It was 96.77% accurate on the test.set, with the parameters I used. I have a lot to learn about the parameters. I also considered eliminating some of the variables (both models ranked variables, and there appeared to be 7-15 variables which were relatively unimportant). However, given the accuracy of this model, I think I am going to stop here. I didn’t really even need to use the validation set I made.
